일별 상담 기록은 월간·분기 요약과 반복 이슈 분석으로 연결했으며, 해당 파일럿 방식은 이후 본사의 Google Sheets 기반 응대 케이스 정리 체계를 추진하는 참고 구조로 확장되었습니다. 공개 페이지에는 실제 고객정보가 아닌 익명화된 구조와 샘플만 사용했습니다.

CASE

분류 기준이 없던 상담 기록을 운영 데이터로 전환

기술지원, 학습문의, 사용문의, 기타의 4개 대분류 아래 총 16개 세부 코드를 설계했습니다. 상담자가 같은 기준으로 기록할 수 있도록 문의자 유형, 문의 내용, 조치 내용, 이관 부서, 완료 상태를 하나의 흐름으로 묶었습니다.

  • 상담 유형을 TS·LI·GI·OH 코드로 구분해 반복 문의를 집계 가능한 데이터로 전환
  • 일별 기록을 카테고리별 집계, 원격지원 분석, 월간 요약, 분기별 요약으로 연결
  • 개인 메모 중심 기록을 팀 단위 공유와 FAQ·SOP 후보 발굴이 가능한 구조로 개선
  • 실제 운영 방식은 이후 본사의 Google Sheets 기반 응대 케이스 정리 체계에 참고·확장

※ 실제 고객명, 캠퍼스명, 연락처, 상세 장애 내용은 공개하지 않고 구조와 익명화 샘플만 사용

MVP

CS/VAN Operations Knowledge Hub MVP

반복 문의, 장애 대응 절차, FAQ 후보, 업무 개선 아이디어를 등록·검색·분류할 수 있도록 설계한 React 기반 지식관리 앱 프로토타입입니다.

  • 문서 등록 / 수정 / 삭제
  • 제목·본문·태그 기반 검색
  • CS 운영, VAN 운영, FAQ, 장애대응, 업무개선 카테고리 분류
  • localStorage 기반 샘플 데이터 저장

※ 실제 회사 시스템이나 실무 데이터를 사용하지 않은 포트폴리오용 MVP입니다.

데이터 구조 연결

상담 데이터
일별 기록의 분류 코드, 문의자 유형, 조치 내용, 이관 부서, 완료 상태를 카테고리별 집계와 월간·분기 요약으로 연결했습니다.
KPI 항목
상담 유형별 발생 건수, 문의자 유형, 이관·완료 현황과 거래 성공률, 실패율, 취소율, 정산 이슈 등 운영 데이터를 KPI 구조로 정리했습니다.
표현 기준
목적은 실제 시스템 구축이나 SQL 활용 역량 과시가 아니라, CS/VAN 운영 이슈를 재사용 가능한 지표 화면과 의사결정 지원 구조로 전환하는 가능성을 검증하는 것입니다.
앱 구조
Knowledge Hub MVP는 현재 localStorage 기반으로 동작하지만, 데이터 처리 로직을 별도 서비스 파일로 분리하여 향후 Supabase API 연동과 로그인 구조 확장을 고려했습니다.

역할과 도구

역할
상담 유형 코드 체계 설계, 일별 기록 필드 정의, 월간·분기 집계 구조화, FAQ/Q&A·매뉴얼 작성, KPI 구조 설계, 지식관리 앱 프로토타입 기획
도구
Excel, Google Sheets, Microsoft Office, GPT, HTML/CSS, JavaScript, React/Vite, Recharts, GitHub, Vercel
방식
현업 상담 흐름을 바탕으로 분류 기준, 기록 필드, 집계 방식, 문구와 샘플 데이터는 직접 설계하고, 구현 과정에서는 AI-assisted workflow로 화면 구조와 기능 흐름을 빠르게 검증했습니다.
보안 기준
실제 회사 데이터, 고객정보, 사업장명, 승인번호, 카드번호, 가맹점번호는 사용하지 않았으며, 공개 가능한 익명화 샘플 데이터만 사용했습니다.

고객 문의를 단순 처리로 끝내지 않고,
분류 기준 → 운영 데이터 → FAQ·SOP·KPI·지식관리로 연결합니다.
