# MK GitHub Pages Portfolio

CS/VAN 운영 경험을 기반으로 업무 흐름, 반복 문의, 결제 운영 이슈를 구조화하고
이를 KPI, 문서화, 화면 설계, 대시보드 MVP로 연결한 개인 포트폴리오입니다.

이 포트폴리오는 전담 개발자 포트폴리오가 아니라,
현업 경험을 바탕으로 업무를 정리하고 시각화할 수 있는 실무형 운영·기획·시각화 역량을 보여주는 것을 목표로 합니다.

## Portfolio Direction

이 포트폴리오는 다음 역량을 중심으로 구성했습니다.

* CS 상담 및 원격지원 경험
* VAN 대리점 전산·결제 업무 이해
* 거래, 정산, 보류정산, 운영비용, 사업장 이슈 흐름 이해
* 반복 문의와 업무 이슈를 FAQ, 문서화, KPI 구조로 정리하는 능력
* 운영 데이터를 화면 구조와 대시보드 형태로 시각화하는 기획 경험
* HTML/CSS 기반 정적 포트폴리오 제작 및 GitHub Pages 배포 경험
* React, Vite, Recharts 기반 KPI Dashboard MVP 제작 및 Vercel 배포 경험
* 뉴미디어디자인 전공 기반 UI/UX, 웹디자인, 데이터 시각화 역량 강화
* AI 도구를 활용한 리서치, 문서화, 화면 설계, 코드 제작 보조 경험

## Featured Project

### VAN Operations KPI Dashboard

VAN 대리점 운영 업무 경험을 기반으로 거래 횟수, 거래 금액, 보류정산, 운영비용, 사업장별 이슈를 KPI로 구조화하고 시각화한 React 기반 대시보드 MVP입니다.

이 프로젝트는 실제 업무 데이터를 사용하지 않고, 공개 가능한 익명화 샘플 데이터만 사용했습니다.

* Store A~G 형태의 샘플 사업장 데이터 구성
* 거래 횟수, 거래 금액, 보류정산, 운영비용, 사업장 이슈 KPI 구성
* 카드, 차트, 표 기반의 운영 대시보드 화면 설계
* React / Vite / Recharts 기반 MVP 제작
* Vercel 배포

Dashboard: https://van-operations-kpi-dashboard.vercel.app

## Data Privacy

본 포트폴리오와 대시보드 프로젝트에는 실제 회사 데이터, 실제 사업장명, 가맹점번호, 승인번호, 카드번호, 사업자번호, 전화번호, 주소, 담당자명, 실제 엑셀 원본 데이터를 사용하지 않았습니다.

공개용 포트폴리오에는 Store A~G 형태의 익명화 샘플 데이터만 사용했습니다.
