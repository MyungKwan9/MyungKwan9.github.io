MVP

CS/VAN Knowledge Hub MVP

CS/VAN 운영 과정에서 반복되는 문의, 장애 대응 절차, FAQ 후보, 업무 개선 아이디어는 시간이 지나면 개인 메모나 산발적인 문서로 흩어지기 쉽습니다. 이를 검색 가능한 지식관리 구조로 정리하기 위해 React 기반의 CS/VAN Knowledge Hub MVP를 제작했습니다.

  • 문서 등록, 수정, 삭제가 가능한 지식관리 앱 프로토타입
  • 제목, 요약, 본문, 태그, 카테고리 기반 검색 구조
  • CS 운영, VAN 운영, FAQ, 업무개선, 장애대응, 매뉴얼 카테고리 분류
  • localStorage 기반 샘플 데이터 저장 및 새로고침 유지
  • 데이터 처리 로직을 articleService로 분리해 향후 Supabase 연동을 고려

이 프로젝트는 실제 운영 시스템이 아니라, 반복 문의를 FAQ 후보로 전환하고 장애 대응 지식을 카테고리별로 축적하는 흐름을 실험하기 위한 프론트엔드 MVP입니다.

React Vite LocalStorage Responsive UI Knowledge Management

※ 본 프로젝트는 실제 회사 시스템이나 실무 데이터를 사용하지 않은 포트폴리오용 프로토타입입니다. 모든 데이터는 샘플이며, 현재 버전은 브라우저 localStorage 기반으로 동작합니다.

데이터 구조 연결

학습 연결
SQLD 준비를 통해 데이터 모델링과 기본 조회 구조를 학습하며, 이를 VAN 결제 운영 KPI 대시보드의 데이터 항목 설계와 지표 구조화에 연결하고 있습니다.
KPI 항목
거래 성공률, 실패율, 취소율, 정산 이슈, VOC 유형 등 운영 데이터를 익명화 샘플 데이터 기반의 KPI 구조로 정리했습니다.
표현 기준
목적은 실제 시스템 구축이나 SQL 활용 역량 과시가 아니라, CS/VAN 운영 이슈를 재사용 가능한 지표 화면과 의사결정 지원 구조로 전환하는 가능성을 검증하는 것입니다.
앱 구조
Knowledge Hub MVP는 현재 localStorage 기반으로 동작하지만, 데이터 처리 로직을 별도 서비스 파일로 분리하여 향후 Supabase API 연동과 로그인 구조 확장을 고려했습니다.

역할과 도구

역할
문의 유형 정리, FAQ/Q&A 구조화, 매뉴얼 작성, KPI 구조 설계, CS 게시판 운영 구조 정리, 지식관리 앱 프로토타입 기획
도구
Excel, Microsoft Office, GPT, HTML/CSS, JavaScript, React/Vite, Recharts, GitHub, Vercel
방식
기획, 정보 구조, 문구, 샘플 데이터 설계는 직접 수행하고, 구현 과정에서는 AI-assisted workflow로 화면 구조와 기능 흐름을 빠르게 검증했습니다.
보안 기준
실제 회사 데이터, 고객정보, 사업장명, 승인번호, 카드번호, 가맹점번호는 사용하지 않았으며, 공개 가능한 익명화 샘플 데이터만 사용했습니다.

고객 문의와 운영 이슈를 단순 처리로 끝내지 않고,
재사용 가능한 개선 자산으로 구조화합니다.
