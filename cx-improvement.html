KPI·운영 리포트 구조화

  • 거래 성공률, 실패율, 취소율, 정산 이슈, VOC 유형을 KPI 항목으로 구조화

MVP

CS/VAN Operations Knowledge Hub MVP

반복 문의, 장애 대응 절차, FAQ 후보, 업무 개선 아이디어를 등록·검색·분류할 수 있도록 설계한 React 기반 운영 지식관리 앱 프로토타입입니다.

  • 문서 등록 / 수정 / 삭제
  • 제목·본문·태그 기반 검색
  • CS 운영, VAN 운영, FAQ, 장애대응, 업무개선 카테고리 분류
  • localStorage 기반 샘플 데이터 저장

※ 실제 회사 시스템이나 실무 데이터를 사용하지 않은 포트폴리오용 MVP입니다.

데이터 구조 연결

학습 연결
SQLD 준비를 통해 데이터 모델링과 기본 조회 구조를 학습하며, 이를 VAN 결제 운영 KPI 대시보드, 지식관리 화면, 운영 리포팅 항목 설계와 연결하고 있습니다.
KPI 항목
문의 유형, 처리 현황, 정산 이슈, 고객사 요청, 운영 비용 등 현업 운영 데이터를 익명화 샘플 데이터 기반의 KPI 구조로 정리했습니다.
표현 기준
목적은 실제 시스템 구축이나 SQL 활용 역량 과시가 아니라, CS/VAN 운영 이슈를 재사용 가능한 문서, 리포트, 지표 화면, 의사결정 지원 구조로 전환하는 가능성을 검증하는 것입니다.
앱 구조
Knowledge Hub MVP는 현재 localStorage 기반으로 동작하지만, 데이터 처리 로직을 별도 서비스 파일로 분리하여 향후 고객사 운영, 영업지원, 지식관리 화면으로 확장 가능한 구조를 고려했습니다.

역할과 도구

역할
문의 유형 정리, FAQ/SOP 구조화, 운영 매뉴얼 작성, KPI 구조 설계, CS 게시판 운영 구조 정리, 지식관리 앱 프로토타입 기획
도구
Excel, Microsoft Office, GPT, HTML/CSS, JavaScript, React/Vite, Recharts, GitHub, Vercel
방식
기획, 정보 구조, 문구, 샘플 데이터 설계는 직접 수행하고, 구현 과정에서는 AI-assisted workflow로 문서, 화면 구조, 코드 초안, 기능 흐름을 빠르게 검증했습니다.
보안 기준
실제 회사 데이터, 고객정보, 사업장명, 승인번호, 카드번호, 가맹점번호는 사용하지 않았으며, 공개 가능한 익명화 샘플 데이터만 사용했습니다.

FAQ·SOP·리포트·지식관리 화면으로 재사용 가능한 개선 자산으로 구조화합니다.
